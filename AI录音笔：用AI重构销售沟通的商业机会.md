## 前言 · 背景说明

新联合众（Linxee）是一家成立于 2011 年、拥有 14 年积累的智能硬件 ODM 企业，现有产品线涵盖音视频类（会议麦克风音箱、会议一体机、USB 摄像头）、存储类、数据转换类及 AI 解决方案（AI 存储精灵、AIDock）。公司具备从工业设计、硬件研发、端侧软件到量产交付的全链路能力，是联想等一线品牌的供应商，获"专精特新"及"高新技术企业"认定。

**现有业务的增长天花板：**

- ODM 模式毛利薄（10-20%），纯硬件差异化空间收窄
- 一次性交易，无持续收入，客户资产沉淀在品牌商而非我们手中
- AI 探索已有基础（AIDock/AI 存储），但软件商业化路径尚未跑通

**AI 重构机会：**

AI 录音笔是最自然的切入点——

- 复用现有音频硬件能力（麦克风阵列、降噪算法、音频处理）
- 延伸已有 AI 落地经验（从 AI 存储 → AI 语音）
- 打开软件订阅收入，从"卖硬件"升级为"硬件入口 + AI SaaS"

我们不是从零造一个新东西，而是把已有的硬件制造能力叠加 AI 软件层，开辟一条毛利更高、用户关系更深的第二增长曲线。

## 一、市场判断

### 1.1 核心结论

AI 录音笔的竞争，不在于"录音"和"转写"本身，而在于能否把语音转化为业务动作，沉淀为组织资产。

2025 年下半年到 2026 年初，钉钉 A1、飞书 × 安克录音豆、GetSeed 录音卡集中爆发，赛道已过概念验证期，但行业心智尚未定型——这是最后的窗口期。

### 1.2 竞品一句话定位

| 竞品 | 定位 | 短板 |
|---|---|---|
| 钉钉 DingTalk A1 | 录音 → 纪要 → 钉钉协同闭环 | 深度业务动作不足，强依赖钉钉生态 |
| 飞书 × 安克录音豆 | 录音 → 妙记 → 待办 → 多维表 | 强依赖飞书生态，跨系统能力受限 |
| GetSeed 录音卡 | 记录 → 整理 → 个人知识生长 | 企业级系统流转和团队管理弱 |
| **我们** | 录音 → 结构化 → 业务系统自动回填，平台中立 | 品牌知名度待建立 |

### 1.3 差异化定位

```
平台中立 × 业务闭环 × 自研硬件 × 开放 SDK 生态 × 无感硬件交互
```

- **平台中立**：不绑定任何单一协同生态（钉钉 / 飞书），数据流向客户自己的系统（CRM / OA / 行业系统）
- **业务闭环**：录音直接触发业务动作，不止于生成纪要
- **自研硬件**：硬件自研制造，成本可控、迭代更快
- **开放 SDK 生态**：AI 语音能力对外授权，从产品公司升维为能力平台（详见 2.4）
- **无感硬件交互**：手势触发 + 场景感知，面谈全程不打断客户节奏（详见 2.5）

## 二、核心价值与重点场景

### 2.1 聚焦：销售面谈

销售面谈是最适合 AI 录音笔切入的场景，同时具备：

**高频发生 · 信息密度高 · 结果导向强 · 可连接 CRM · ROI 可量化**

### 2.2 现状痛点

一个 100 人规模的销售团队，每人每天平均 2-3 次客户沟通，但 CRM 有效填写率不到 25%。

**为什么会这样：**

- 拜访结束靠回忆手填，30 分钟后细节留存率急剧下降
- 繁忙时直接跳过，填写质量参差不齐
- 竞品提及、价格异议等关键信号没有系统性捕捉机制
- 优秀销售的话术和判断停留在个人经验，无法组织沉淀

**管理者同样看不到：**

看板上只有通话时长和阶段标记，无法还原一线沟通实况——客户说了什么、提了哪些异议、哪个客户有竞品风险——完全不可见。

### 2.3 我们的解决路径

```
录音  →  转写 + 角色识别  →  结构化提取  →  系统回填  →  跟进行动
低打扰开启     多人分离       需求/异议/      CRM 自动填写    跟进提醒
稳定收音       重点标记       承诺/下一步     主管看板更新    话术复盘
```

- **对销售来说**：拜访完确认一下直接推送，30 秒收尾，不用再手填 CRM。
- **对主管来说**：每个客户的真实沟通情况实时可见，竞品风险 AI 自动标红。

### 2.4 差异化价值一：开放 SDK / API + 行业 Skill Pack

> **竞品都是"封闭管道"，我们是"开放能力层"。**

#### 开放 SDK / API 平台

现有竞品的软件架构全部是封闭生态：

- 钉钉 A1 → 只能回填钉钉
- 飞书录音豆 → 只能回填飞书妙记
- GetSeed → 无对外 API

**我们的做法**：将核心 AI 能力（转写 API、多角色分离 API、结构化提取 API）对外开放授权，让 CRM 厂商（纷享销客、Salesforce）、HIS 医院系统、OA 平台直接调用 SDK 集成，而不是让用户手动"推送"过去。

这本质上是从"产品公司"升维为"AI 语音基础设施提供商"——类似 Twilio 在通信领域的地位：**不卖通话，卖通话能力**。集成越多，客户切换成本越高，护城河随网络效应自然加深。

这一方向在商业模式中已预留路径（见 4.4）；2.4 是其产品层面的落地方式。

#### 行业 Skill Pack（场景技能包）

将各行业的 AI 提取逻辑封装为可下载的"Skill 包"，类似 App Store，但运行在 AI 能力层：

| Skill 包 | 核心提取内容 | 目标用户 |
|---|---|---|
| 销售面谈 Skill | 需求 / 异议 / 竞品风险 / 承诺项 | B2B 销售团队 |
| 医疗问诊 Skill | 主诉 / 现病史 / 诊断意见 | 诊所、医院 |
| 法律会见 Skill | 争议焦点 / 证据引用 / 时间线 | 律师事务所 |
| 内部会议 Skill | 决议 / 待办 / 负责人 / Deadline | 企业团队 |
| 客服回访 Skill | 投诉分类 / 情绪识别 / 处理结果 | 客服团队 |

用户买一台设备，根据当天场景切换 Skill，一个硬件覆盖多个工作流。Skill 包可以单独定价（行业包 2,000-5,000 元/年），也可开放第三方开发者共建，形成"AI 语音生态"——这是任何单一封闭竞品无法复制的能力。

### 2.5 差异化价值二：无感硬件交互——手势触发 + 场景感知自动开录

> **现有录音笔都需要"有意识地去操作设备"，这在真实销售面谈场景里是反人性的。**

**当前痛点**：销售拜访客户时，掏出设备、手动按键启动录音这个动作，会：

- 打断自然的对话节奏
- 引发客户对"录音"的敏感警觉
- 导致很多关键开场信息来不及录到

#### 创新点一：双击体感触发（Tap-to-Record）

设备放在夹克口袋或桌面，用手指轻敲外壳两次（无需找按键、无需看屏幕）→ 加速度传感器识别双击振动模式 → 静默开始录音 → 以一次轻微震动反馈确认已启动。

**全程 0.5 秒、无屏幕亮起、无提示音**，谈话对象完全无感知。这是销售场景的"隐形开关"，也是产品的硬件体验记忆点。

#### 创新点二：场景感知自动待录

比传统"语音唤醒词"更进一步，不需要说出任何指令：

- **双声源感知启动**：麦克风阵列检测到"两个人交替说话"的声学特征（音量、频率交替变化超过阈值），自动进入静默录音待机状态，无需任何手动操作
- **日历 + 位置联动预热**：与手机日历同步，检测到"今天有客户拜访日程"且 GPS 到达客户公司附近时，设备自动唤醒、提前预热 ASR 模型——人坐下开口就已经在录了

#### 与竞品硬件交互对比

| 产品 | 开录方式 | 面谈场景适应性 |
|---|---|---|
| 钉钉 A1 | 手动按键 / App 操作 | 需要明显操作，影响对话 |
| 飞书录音豆 | 物理按键 | 同样明显 |
| GetSeed 录音卡 | 夹在衣领，仍需手动激活 | 稍好，但无智能感知 |
| **我们** | 双击体感 + 声学场景感知 | **全程无感，不打断沟通节奏** |

## 三、功能与交互设计

> 关键界面线框图详见配套原型文件 [prototype.html](./prototype.html)，以下为文字说明。

### 3.1 核心交互流程

```
【会前】              【会中】              【会后】            【跟进】
开启设备        →     正常沟通           →  查看纪要       →   补充备注
选择场景模板          实时转写               确认结构化信息       推送 CRM
                    关键词捕捉              一键同步系统         触发提醒
                    ⚠ 风险实时提示                             主管看板更新
```

### 3.2 界面 A：录音结束 → 纪要自动生成

用户结束拜访，打开 App，AI 已在后台完成转写和初步分析。用户无需等待，直接看到"已在处理"的状态和摘要预览。

**设计说明：**

- 录音基本信息（时长、时间、场景标签）一目了然，给用户"这次拜访被完整记录"的安全感
- AI 生成状态实时可见，摘要先行露出，降低用户等待焦虑
- 两个出口清晰：继续录音 or 查看完整纪要，不打断用户后续安排

### 3.3 界面 B：结构化信息卡片

这是产品的核心"哇时刻"——用户看到 AI 把一段 23 分钟的对话，自动拆解成了客户需求、异议点、承诺项、下一步行动等结构化信息。

**设计说明：**

- Tab 切换设计，4 类信息分开呈现，不堆叠，符合销售的使用习惯（"我只想看异议点"）
- 每条信息带置信度标签（核心需求 / 隐性需求 / 价格敏感），让用户知道 AI 判断的依据
- 橙色竞品风险底部提示，自动标记，主管端同步可见
- AI 建议以"参考项"呈现，用户可修改，不强制写入——AI 提取，人来判断

### 3.4 界面 C：一键推送 CRM 确认

用户确认结构化信息后，一键推送至企业 CRM 系统。

**关键设计**：字段映射透明可见，用户可修改后再同步，不强制覆盖。

**设计说明：**

- 字段映射预览解决"AI 写进去的我不知道对不对"的信任问题
- 竞品风险字段橙色高亮，提醒用户重点关注
- "可手动修改后再同步"降低用户顾虑，避免错误数据污染 CRM
- 提供"仅保存本地"的逃生出口，强调用户掌控权

### 3.5 其他场景简览

以销售面谈为切入点，后续可延伸至：

| 场景 | 核心价值 | 对接系统 |
|---|---|---|
| 医疗问诊 | 结构化病历自动生成，医生确认 + 修改，从"打字员"回归"医生" | HIS（医院信息系统） |
| 法律会见 | 争议焦点 / 证据引用自动提取，归档可检索 | 案件管理系统 |
| 内部会议 | 决议待办自动分发，知识资产可检索 | OA / 飞书 / 钉钉 |

## 四、商业模式

### 4.1 卖给谁

| 层级 | 目标客户 | 购买动机 |
|---|---|---|
| 核心客户 | 销售团队、保险/金融顾问团队、企业服务团队、招商/渠道团队 | 提高记录质量、提升转化效率、降低管理成本 |
| 扩展客户 | 培训机构、咨询团队、律师顾问、猎头访谈、调研团队 | 高强度口头信息整理与沉淀 |
| 后续延展客户 | 医疗访谈、政企服务、门店巡检等行业 | 行业知识提炼与流程标准化 |

### 4.2 怎么卖

增长路径：App 免费体验 → 个人付费 → 团队采购 → 行业定制

```
冷启动阶段             规模化阶段              利润阶段
──────────      ──────────          ──────────
手机 App 免费体验       团队版 PLG 扩散           行业解决方案
↓                    ↓                       ↓
用了回不去             个人 → 公司内传播          医疗 / 法律 / 金融
↓                   → IT 部门采购团队版        定制 + 私有化部署
硬件自然升级
（App → 录音笔）
```

**关键策略：免费版卡时长不卡功能**

所有 AI 能力免费体验，只限制用量（300 分钟/月）。让用户充分感受价值后，为"更多时长"付费，转化阻力最小。

### 4.3 核心商业化策略

| 策略 | 具体做法 |
|---|---|
| 硬件补贴获客 | 硬件毛利控制在 10% 以内甚至平价，目标是设备激活量；企业批量采购可赠送硬件（绑定年订阅） |
| PLG 增长飞轮 | 【阶段一·无硬件轻启动】用户通过手机 App 免费体验完整 AI 能力，零门槛传播；【阶段二·硬件升级转化】用户遇到手机录音瓶颈（收音距离/续航/多人分离）时，以"效果升级"驱动购买录音笔 → 达到团队临界点后 IT 采购团队版 |
| 免费版卡时长不卡功能 | 所有 AI 能力免费体验，只限制用量（300 分钟/月）——让用户充分感受价值后，为"更多时长"付费，转化阻力最小 |
| 行业版做标杆案例 | 每个垂直行业投入资源做 1-2 个标杆客户，产出 ROI 数据（如"病历书写时间减少 60%"），用案例撬动同行业批量复制 |
| 开放集成做渠道 | 与 CRM 厂商（纷享销客/Salesforce）、HIS 厂商、OA 厂商做集成认证，形成"中立连接器"定位——不与任何平台形成排他关系，借助多方渠道获客；集成越多，客户越难替换 |

### 4.4 收入结构

| 来源 | 占比 | 定价 | 核心逻辑 |
|---|---|---|---|
| 硬件销售 | 15-20% | 基础款 399-499，旗舰款 699-899 | 微利获客，不靠硬件赚钱 |
| 个人/团队订阅 | 50-60% | 29-199 元/人/月 | 核心现金流，标准 SaaS，毛利 80%+ |
| 行业解决方案 | 20-30% | 10 万-100 万+/年 | 壁垒与利润并重，标杆案例撬动同行 |

**未来延伸**：待自有产品 AI 能力成熟后，可向其他硬件品牌商开放 SDK 授权和 API 调用，从"产品公司"进化为"AI 语音能力平台"。

## 附录 0：产品团队的中台角色

> 我们是软件产品团队，偏市场前端——不是纯研发，也不是纯销售，而是在两侧之间做翻译和串联。

### 向研发侧输出：产品需求的边界定义

| 研发常见问题 | 产品中台给出的答案 |
|---|---|
| AI 准确率要到多少才能上线？ | 结构化提取置信度低于 70% 的字段，进入"待确认"状态，不自动写入 CRM；让用户判断，而非让 AI 自决 |
| 端侧还是云端处理？ | 实时字幕用端侧轻量 ASR（低延迟）；结构化提取用云端大模型（高精度）；行业版支持私有化部署 |
| 硬件双击触发的误触怎么处理？ | 定义触发条件：两次敲击间隔 200-500ms，且设备当前处于静止状态（陀螺仪辅助判断）；误触概率 < 0.5% |
| Skill Pack 怎么版本管理？ | Skill 包与 App 版本解耦，独立更新；用户可以回退 Skill 版本；企业版 Skill 锁定在 IT 审核通过的版本 |

### 向销售侧输出：可量化的销售 Pitch

销售面对客户时最常被问到：**"这个产品对我们有什么用？能算出 ROI 吗？"**

产品团队需要提前把答案算好：

```
以 100 人销售团队为例：
• 当前 CRM 有效填写率 25% → 目标提升至 75%
• 平均每人每天节省手填时间 20 分钟
• 100 人 × 20 分钟 × 250 个工作日 = 每年节省 8,333 小时
• 按人效成本折算（销售平均时薪约 60 元）→ 年节省约 50 万元人力成本
• 团队版定价 99 元/人/月 → 年成本 11.88 万元
• ROI ≈ 4.2x，回收周期约 3 个月
```

这套数字由产品团队算清楚，交给销售当武器，不让销售自己去解释 AI 怎么工作。

## 附录 1：AI 能力边界说明

我们知道 AI 在哪里会出问题——这比知道 AI 能做什么更重要。

| 边界 | 应对设计 |
|---|---|
| 转写不稳定（方言/噪音/多人交叉） | 硬件侧定向麦克风降噪，低置信度片段标记提示人工校对 |
| 结构化可能有偏差 | 高价值字段（决策/承诺项）提供人工确认步骤，不自动写入 |
| AI 无法替代业务判断 | AI 建议以"参考项"呈现，人负责判断和决策 |
| 实时性与数据合规的张力 | 端侧轻量 ASR 负责实时字幕，云端大模型异步处理；行业版支持私有化部署 |

## 附录 3：AI 数据飞轮机制

这是 AI 产品区别于传统软件的核心：**产品越用越准，形成竞品难以追赶的时间壁垒。**

```
用户使用产品
     ↓
AI 提取结构化信息
     ↓
用户修改 / 确认 / 删除某些字段   ←── 这是有标注的训练信号
     ↓
行业专属微调数据积累
     ↓
行业 Skill 模型精度提升
     ↓
用户感知到"AI 越来越准"
     ↓
用户留存提升 · 口碑传播
     ↓
更多用户使用 → 循环
```

**关键设计原则**：每次用户编辑 AI 提取结果的行为，不只是"纠错"，而是在生产高质量的行业标注数据。这些数据只属于我们，无法被竞品购买或复制。

用户在销售场景里积累的数据，不能用来训练医疗模型——数据是行业隔离的，这也保证了客户数据合规和安全。
